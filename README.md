# MOT Training Cooling

MOT 研修で用いる Cooling ビームの評価・整形条件の検討用ノートブック集です。

このディレクトリでは、実測したビーム径データをもとに

- ガウシアンビームとしての初期パラメータを推定する
- レンズ通過後のビーム伝搬を ABCD 行列で追跡する
- レンズ位置と焦点距離の候補を比較し、実験で使いやすい条件を絞り込む

ことを目的にしています。

## Overview

MOT 用レーザーは理想的な円形ビームではなく、実際には

- x 方向と y 方向でビームサイズが異なる
- ウェスト位置がずれる
- 非点収差の影響がある

ことが多くあります。

そのため、このリポジトリでは x, y を独立に扱いながら、

1. 実測データの Gaussian fitting
2. q パラメータによる自由空間伝搬とレンズ通過の記述
3. 実験空間で使える有効長の評価
4. 現実的なレンズ配置の選定

を順番に行います。

## Files

### `Beam_Gaussian_Fitting.ipynb`

メインの設計ノートブックです。

- 測定点から x, y 方向のビームウェスト、ウェスト位置、レイリー長を推定
- レンズ位置 `a` と焦点距離 `f` を変数として走査
- レンズ後のウェスト位置、ウェスト半径、レイリー長を計算
- x, y 両方向で実験に使える有効長を比較
- 非点収差とビーム形状のバランスから候補条件を選定

研修で「最終的にどの条件を採用するか」を考えるときの中心ファイルです。

### `waist_0418.ipynb`

実測ビームから出発して、よりシンプルな光学系でビーム伝搬を確認するノートブックです。

- 出射ビームの x, y 半径データを Gaussian fitting
- q パラメータを定義
- 1 枚レンズを入れた場合の変化を計算
- 2 枚レンズ構成にした場合の変化を計算

研修で「まずビームがどう進むか」をつかむのに向いています。

### `ノートブック解説.md`

各ノートブックの各セルが何を計算しているかを、研修向けに文章で説明した解説ファイルです。

以下のような確認に役立ちます。

- このセルの入力値は何か
- この式は何を意味しているか
- どの図を見ればよいか
- x と y を分けている理由は何か

## Main Physical Quantities

このリポジトリでは、以下の量が重要です。

### Beam waist `w0`

ビームが最も細くなる位置での半径です。

### Waist position `z0`

ビームウェストが存在する位置です。

### Rayleigh length `zR`

ウェスト近傍で、ビームが大きく広がらず扱いやすい範囲の目安です。

### q parameter

複素ビームパラメータ

```text
q = -z0 + i zR
```

として表され、自由空間伝搬やレンズを ABCD 行列でまとめて扱うために使います。

### Astigmatism

x 方向と y 方向でウェスト位置が一致しないことによるずれです。

ファイバーカップリングやビーム整形では、このずれが大きいと不利になることがあります。

## Typical Workflow

このリポジトリでの基本的な流れは次の通りです。

1. 測定したビーム径を x, y 方向それぞれ入力する
2. Gaussian fitting により `w0`, `z0`, `zR` を求める
3. q パラメータを作る
4. 自由空間伝搬とレンズ通過を ABCD 行列で計算する
5. 候補となるレンズ位置と焦点距離を比較する
6. 有効長、非点収差、ビーム形状のバランスから採用条件を決める

## Notes

- x と y は別々に扱っています。ビームが円形でない場合、片方だけ見ても十分ではありません。
- ノートブック中の距離単位は `m`, `mm`, `um` が混在するため、単位換算に注意してください。
- 実験で採用する条件は、理論上の最適値だけでなく、手持ちのレンズや設置スペースも考慮して決めます。
- ノートブック内の候補比較では、非点収差の小ささとビーム長短比の両方を見ています。

## For Training

研修では、次の観点で読むと理解しやすいです。

- なぜ x と y を分けてフィットするのか
- レイリー長を「使いやすい範囲」の指標にしているのはなぜか
- q パラメータを使うと何が楽になるのか
- ビームを円形に近づけることと、非点収差を減らすことの間にどんなトレードオフがあるか
