# PathB.ipynb ロジック引継ぎ

この資料は、`PathB.ipynb` が何を入力にして、どのような順番で計算し、どの指標で経路Bの候補を評価しているかを、テキストベースで引き継ぐための資料です。

---

## 1. この notebook の目的

`PathB.ipynb` の目的は、経路Bに対して

- 2 枚レンズ系を通した後のビームがどうなるか
- AOM 面でビームが開口に収まるか
- カップラー位置でどのようなビーム径になるか
- カップラーレンズ通過後の発散角が SM ファイバーに合うか
- 最終候補同士を比べたとき、非点収差がどの程度あるか

を評価し、実際に組む候補を絞ることです。

この notebook は、

- `Beam_Gaussian_Fitting.ipynb` の Cell 7 で得た初期ビーム
- `OptimizeDivergenceAngle.ipynb` の 2 枚レンズ + カップラーの考え方

を統合した notebook です。

---

## 2. 前提としている考え方

### 初期ビームは再フィットしない

この notebook では、初期ビームは別 notebook で既に求まっているものとします。

つまり、

- `Beam_Gaussian_Fitting.ipynb` の Cell 7 で求めた `w0_x, z0_x, w0_y, z0_y`

を固定入力として使います。

理由は、今回の経路Bで使うビームはその notebook で扱ったビームと同じだからです。

### レンズ配置評価は q パラメータで行う

レンズ配置の検討は、ガウシアンビームの `q パラメータ` を使って行います。

具体的には、

- 自由空間伝搬: `q -> q + d`
- 薄レンズ通過: `q -> 1 / (1/q - 1/f)`

で計算しています。

### AOM とカップラーを同時に考える

経路Bでは、単にビームが細くなるだけでは不十分です。

同時に、

- AOM 開口に収まるか
- カップラー位置で適切なビーム径か
- カップラーレンズ通過後の発散角が SM ファイバー条件に近いか

を見る必要があります。

## 3. notebook の入力

### 3.1 初期ビームパラメータ

`Beam_Gaussian_Fitting.ipynb` の Cell 7 の結果を使っています。

- `LAM = 852.3e-9`
- `W0_X = 162.6e-6`
- `Z0_X = -101.8e-3`
- `W0_Y = 334.7e-6`
- `Z0_Y = -24.7e-3`

ここで、

- `W0_X`, `W0_Y` は x / y 方向のウェスト半径
- `Z0_X`, `Z0_Y` は x / y 方向のウェスト位置

です。

### 3.2 AOM 規格

今回の AOM 規格を notebook 内に固定値として入れています。

- 使用波長範囲: `780-850 nm`
- 動作周波数: `200 MHz`
- 開口: `0.32 mm`
- Rise/Fall Time: `10 ns`

この notebook の幾何光学評価で直接効いているのは主に `AOM_ACTIVE_APERTURE = 0.32 mm` です。

### 3.3 ファイバー条件

SM ファイバーの MFD を

- `SM_MFD = 5.3 um`

として入れています。

これを使って、

- `SM_TARGET_DIVERGENCE = λ / (π * (MFD/2))`

で、理想的な発散角の目標値を作っています。

つまり最終的に、カップラーレンズ後の発散角がこの値に近い候補が良いとみなします。

## 6. Cell 11 の主ロジック

### 6.1 `evaluate_path_b(...)` の役割

この notebook の中心関数です。

入力:

- `d1`: lens1 までの距離
- `f1`: lens1 の焦点距離
- `d2`: lens1 から lens2 までの距離
- `f2`: lens2 の焦点距離
- `aom_z`: AOM の位置
- `coupler_z`: カップラーの位置
- `f3`: カップラーレンズの焦点距離

出力:

- lens1, lens2 の位置
- lens2 後の x / y の状態
- AOM 面でのビーム径
- カップラー面でのビーム径
- AOM 開口を満たすかどうか
- AOM 開口に対する充填率
- カップラーレンズ後の発散角
- MFD 由来の目標発散角との差
- lens2 後と coupler 後の非点収差

### 6.2 この関数の計算順序

1. 初期ビームから `q0_x`, `q0_y` を作る
2. lens1 まで自由空間伝搬
3. lens1 を通す
4. lens2 まで自由空間伝搬
5. lens2 を通す
6. lens2 直後の `q2_x`, `q2_y` を得る
7. その `q2` を使って AOM 面とカップラー面のビーム径を計算する
8. AOM 面の最大直径が `0.32 mm` 以下か判定する
9. カップラー位置まで伝搬し、カップラーレンズ `f3` を通す
10. カップラーレンズ後の発散角 `theta_x`, `theta_y` を求める
11. 目標発散角との差を評価する
12. 参考値として非点収差を出す

### 6.3 非点収差の位置づけ

この notebook では、非点収差は主最適化指標ではなく、`参考指標` です。

つまり、

- まず AOM 開口に入るか
- つぎに発散角が目標に近いか
- その後で非点収差を比較する

という順番になっています。

---

## 7. `search_coupler_positions(...)` の役割

この関数は、カップラー位置を走査して候補を並べ替えるための関数です。

処理内容:

1. 指定範囲の `coupler_z` を等間隔に走査
2. 各位置で `evaluate_path_b(...)` を実行
3. AOM 開口条件を満たした候補だけ残す
4. 次の順でソートする

- `theta_error`
- `astigmatism_after_coupler_mm`
- x / y ビーム径差

つまり、

`まず発散角が目標に近い候補を優先し、その中で非点収差と楕円性が小さいものを上に置く`

という並べ方です。

## 10. 出力の見方

### `AOM aperture check`

- `PASS`: AOM 面での最大直径が開口以下
- `FAIL`: AOM 開口を超えている

### `AOM fill ratio`

- `1.0` に近いほど開口いっぱい
- `1.0` を超えると物理的に入らない

### `theta_x`, `theta_y`

カップラーレンズ後の x / y 方向発散角です。

### `theta_target(MFD)`

SM ファイバー MFD から作った目標発散角です。

### `theta_error(max)`

x, y のうち、目標からずれている大きい方です。

この値が小さい方が、SM ファイバー結合には有利とみなしています。

### `astig after lens2`

lens2 後の x / y ウェスト位置差です。

### `astig after coupler`

カップラーレンズ後の x / y ウェスト位置差です。

この notebook では、非点収差は「最終候補比較の補助指標」です。

## 11. この notebook の設計思想

この notebook は、次の優先順位で候補を見ています。

1. まず AOM 開口に収まること
2. 次に SM ファイバーの目標発散角に近いこと
3. そのうえで非点収差が小さいこと
4. 最後に x / y のビーム径差が小さいこと

つまり、

`まず AOM とファイバーに物理的に入る候補を確保し、その中で非点収差の小さいものを選ぶ`

という設計思想です。

これは今回の経路Bの目的に合っています。

## 12. 次に notebook を触る人への注意

### 12.1 初期ビームを変えたら Cell 3 を直す

ビーム測定をやり直した場合は、Cell 3 の

- `W0_X`, `Z0_X`
- `W0_Y`, `Z0_Y`

を更新する必要があります。

### 12.2 AOM を変えたら Cell 5 を直す

特に重要なのは

- `AOM_ACTIVE_APERTURE`

です。

AOM 開口が変わると、通せる候補が大きく変わります。

### 12.3 ファイバーを変えたら Cell 7 を直す

SM ファイバーの MFD が変わると、目標発散角も変わります。

### 12.4 今は AOM 周波数自体は幾何光学評価に直接使っていない

`AOM_OPERATING_FREQUENCY = 200 MHz` は notebook に入っていますが、
今のロジックでは主に記録用です。

この notebook で直接効いているのは主に

- 開口
- レンズ配置
- カップラー位置

です。

### 12.5 非点収差は参考値

今回の notebook では、非点収差を完全に無視してはいませんが、
主指標にはしていません。

本当に最終候補を 1 つに絞るときは、

- 実際のカップリング効率
- AOM 回折効率

も合わせて実測確認する必要があります。
